🚀 خطوات الرفع على Railway:
═══════════════════════════════════════════════════════

1️⃣ إنشاء حساب:
   - اذهب إلى: https://railway.app
   - سجل دخول بحساب GitHub

2️⃣ إنشاء مشروع جديد:
   - اضغط "New Project"
   - اختر "Deploy from GitHub repo"
   - أو اختر "Empty Project" ثم "Deploy from GitHub"

3️⃣ رفع الملفات:
   طريقة 1 - عبر GitHub:
   - ارفع كل الملفات على GitHub Repository
   - اربط الـ Repository مع Railway
   
   طريقة 2 - عبر Railway CLI:
   - حمل Railway CLI
   - نفذ: railway login
   - نفذ: railway init
   - نفذ: railway up

4️⃣ التحقق من الإعدادات:
   - Railway سيكتشف Python تلقائياً
   - سيستخدم Procfile لأمر التشغيل
   - سيثبت المكتبات من requirements.txt

5️⃣ الانتظار:
   - Railway سيبدأ التشغيل تلقائياً
   - انتظر 2-3 دقائق للبناء والنشر
   - ستحصل على رابط التطبيق

═══════════════════════════════════════════════════════
⚙️ الإعدادات التلقائية:
═══════════════════════════════════════════════════════

✓ المنفذ (PORT): يتم تحديده تلقائياً من Railway
✓ مجلد البيانات (data/): يُنشأ تلقائياً عند أول تشغيل
✓ الذاكرة: كافية للتطبيق (512MB - 1GB)
✓ إعادة التشغيل: تلقائية عند حدوث مشاكل

═══════════════════════════════════════════════════════
🔧 استكشاف الأخطاء:
═══════════════════════════════════════════════════════

❌ خطأ "Port already in use":
   → Railway يحدد PORT تلقائياً، لا داعي للقلق

❌ خطأ "Module not found":
   → تأكد من وجود جميع المكتبات في requirements.txt

❌ التطبيق لا يعمل:
   → افتح Logs في Railway Dashboard
   → تحقق من رسائل الخطأ

❌ مشاكل WebSocket/SocketIO:
   → تأكد من أن Railway يدعم WebSocket (مفعّل افتراضياً)
